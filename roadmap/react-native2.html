Your First Creative Sandbox: Expo Go

Your journey begins with a single command: `npx create-expo-app my-app`. This sets up a complete React Native project for you. Then, you download the Expo Go app on your iOS or Android phone. Scan the QR code from your terminal, and your app instantly appears on your phone. It's magic.

Why Is This the Right Place to Start?

The developer experience is everything. Expo removes the single biggest barrier to entry for React Native beginners: native tooling. You don't have to think about simulators, build configurations, or native code. You just write JavaScript in your text editor, save, and the app on your phone reloads instantly. You are learning React Native in the most direct and rewarding way possible, focusing on building UI, not on configuration.

A common fear is, "Styling is weird. It's not CSS." You're right, it's not. It's a JavaScript-based system inspired by CSS. But the properties are almost all the same (`flexbox` is king!), and we'll see how this system makes your styles more maintainable and component-scoped by default.

Your Foundational Mission: A 3-Phase Plan to Confidence

Our mission is not to make you a "senior mobile engineer." That will come with time. Our mission is to make you *capable* and *confident* in building a complete, multi-screen mobile app. We will do this by mastering three core concepts.

Phase 1: Master the Core Components

A mobile app is just a collection of views. Your first goal is to master React Native's fundamental building blocks. In your new Expo project, learn *only* these three things.

  • `` and ``: The `div` and `p`. These are the two most essential components. Learn to use `` to structure your layout and `` to display any and all text. All other components are built from these.
  • `StyleSheet.create()`: The "CSS-in-JS". Learn how to create a stylesheet object to define your component's styles. Master basic properties like `backgroundColor`, `fontSize`, and the core of all mobile layout: `flex`, `flexDirection`, `justifyContent`, and `alignItems`.
  • `Pressable` and `useState`: The Interactivity. Learn how to use the `Pressable` component to detect user taps. Combine this with the `useState` hook from React to create a simple counter or toggle a piece of text.

Your Goal for this Phase: Practice until you can build a single screen with a title and a button that, when pressed, changes the title's text.

Phase 2: Become a Navigation Detective

Now that you can build a screen, let's build an app with multiple screens. The standard for this is a library called React Navigation. Your mission is to build a simple two-screen app: a "Home" screen and a "Details" screen.

This is an exercise in deconstructing app architecture. As you install and configure React Navigation, ask yourself questions a detective would ask:

  • What is a "Stack Navigator"? How does it manage the history of screens I've visited? (It "pushes" new screens onto a stack, which is why you get a "back" button for free.)
  • How do I navigate from the Home screen to the Details screen? (This is using the `navigation.navigate('Details')` function that the library provides.)
  • How can I pass data, like a product ID, from the Home screen to the Details screen? (This is passing "params" in the navigate function: `navigation.navigate('Details', { itemId: 86 })`.)

Your Goal for this Phase: To stop seeing an app as one giant component and start seeing it as a structured collection of screens that you can move between.

Phase 3: The Creator's Leap—Your First Real App Screen

This is the most important phase. You will now transition from a student to a creator. Your mission is to build a simple but realistic "User Profile" screen. It should display a user's profile picture, their name, username, and a short bio.

Why this project? It's a perfect test of your layout and component skills. It forces you to use Flexbox to arrange items both vertically and horizontally. You'll need an `` component for the profile picture, several `` components for the information, and `` components to group them all together. This is a microcosm of almost every screen in every app you've ever used.

Your Goal for this Phase: To create a single, well-designed screen from scratch that looks and feels like a real app. Perfection is not the goal. Completion is.
